# 🤖 インタラクティブ・マルチエージェント開発システム 改善まとめ ＆ 引継ぎガイド

別PC環境への移行後もそのままマルチエージェントシステムを再現・運用できるように、今回の改修内容、システム構成、およびセットアップ・実行手順をまとめています。

---

## 📌 1. システム概要・アーキテクチャ

本システムは、複数分野の最高性能AIモデルを役割分担させ、ユーザーとの対話による計画策定からコード生成・デバッグ検証までを自動化する開発支援システムです。

### 👥 エージェント体制 ＆ 使用モデル
| 役割 | エージェント | 使用モデル | 担当内容 |
| :--- | :--- | :--- | :--- |
| 👑 統括マネージャー | Manager | `antigravity-gemini-3.1-pro-high` | 要件分析、ヒアリング質問生成、計画書策定、最終統括 |
| 🎨 フロントエンド | Front-end | `antigravity-gemini-3-flash` | UI/UX設計、画面構成、フロントエンドコード生成 |
| ⚙️ バックエンド | Back-end | `antigravity-gemini-3.1-pro-high` | データ構造・API設計、ロジック・バックエンドコード生成 |
| 🔍 品質保証・検証 | QA | `antigravity-claude-opus-4-6-thinking` | コードレビュー、バグ・セキュリティ検証、デバッグ報告 |

- **通信基盤**: `ag-local-bridge` エンドポイント (`http://localhost:11435/v1/chat/completions`)

## ✨ 2. 今回改善・実装された新機能

### ① 対話型ヒアリング ＆ 選択肢（+自由記述）システム
- マネージャーがユーザーのプロンプトを受け取った後、仕様決定のための質問を多肢選択肢（1〜N）形式で出力。
- ユーザーは番号（1, 2...）で手軽に答えることも、選択肢にない要望を文章（Other）で自由に指定することも可能です。

### ② 確定計画書の提示 ＆ ユーザー承認確認ステップ
- マネージャーとの議論で「高精度開発計画書」が完成しても、勝手にエンジニアに渡されません。
- 画面上に計画書全体が表示され、**「承認して進める (y / Enter)」** か **「修正指示を入力する」** かを確認します。

### ③ 連続プロンプト入力（メインループ化）
- 1つの機能や実装が完了した後もプログラムが終了せず、次のプロンプト入力待ちに戻ります。
- `exit` または `quit` を入力するまで会話・開発を継続できます。

### ④ エージェント別ログのリアルタイム書き出し
- 各エージェントのログメッセージが `logs/` ディレクトリ配下に個別リアルタイム追記されます。
  - `logs/manager.log`
  - `logs/frontend.log`
  - `logs/backend.log`
  - `logs/qa.log`

### ⑤ tmux による左右画面分割ターミナルUI
- ログがターミナルのスクロールで流れて見えなくなる問題を解決するため、`tmux` による左右ペイン分割を導入。
  - **左ペイン（大）**: マネージャー ＆ ユーザーのメイン対話画面
  - **右上ペイン**: フロントエンドのリアルタイムログ
  - **右中ペイン**: バックエンドのリアルタイムログ
  - **右下ペイン**: QAエンジニアのリアルタイム検証ログ

## 📂 3. ファイル構成

```text

├── start_team_tmux.sh         # tmux 画面分割自動起動スクリプト（Bash）
├── multi_agent_output.md      # 最新の成果物（確定計画書、生成コード、QA報告）

    ├── manager.log
    ├── frontend.log
    ├── backend.log
    └── qa.log
```

## 🚀 4. 新しいPCでのセットアップ・実行手順

### Step 1: 必要なパッケージのインストール
新しいPCのターミナルで `tmux` をインストールします（Linux / WSLの場合）。

```bash
sudo apt-get update
sudo apt-get install -y tmux python3
```

### Step 2: 実行権限の付与
ワークスペースディレクトリに移動し、起動スクリプトに実行権限を与えます。

```bash
cd /path/to/workspace
chmod +x start_team_tmux.sh
```

### Step 3: ローカルブリッジ (`ag-local-bridge`) の確認
ポート `11435` で Antigravity Local Bridge が動作していることを確認してください。

### Step 4: システムの起動

#### 画面分割（tmux）モードで起動する場合【推奨】
```bash
./start_team_tmux.sh
```
*(1つのウィンドウが左右4分割され、右側で各エージェントの動きを監視しながら左側でマネージャーと対話できます)*

#### 通常のターミナル単体で起動する場合
```bash
python3 multi_agent_team.py
```

## 💡 5. 使い方・操作の流れ

1. **起動**: `./start_team_tmux.sh` を実行。
2. **要件入力**: `初期プロンプト >` に作りたい機能やシステムを入力。
3. **ヒアリング回答**: マネージャーの質問に対し、選択肢番号（1, 2...）または自由テキストで回答。
4. **計画承認**: マネージャーから「高精度開発計画書 (案)」が提示されたら内容を確認。
   - 問題なければ `y` または `Enter` を入力。
   - 修正したい場合は修正内容を入力。
5. **コード制作 ＆ QA検証**: 承認後、エンジニアがコードを作成しQAがデバッグ検証を実施。結果が `multi_agent_output.md` に保存されます。
6. **継続・終了**: 次のプロンプトを入力して開発を続けるか、`exit` で終了します。
